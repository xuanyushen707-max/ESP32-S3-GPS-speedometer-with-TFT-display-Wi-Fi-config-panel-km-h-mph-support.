========================================================================
           ESP32-S3 GPS 速度计 + ST7735 TFT 显示屏 v1.0
========================================================================

一、硬件清单
------------------------------------------------------------------------
1. ESP32-S3 DevKitC-1 开发板 x1
2. ST7735 128x128 1.44寸 TFT LCD 显示屏(x1 SPI 接口)
3. ATGM336H GPS 模块 x1
4. USB-C 数据线 (供电+烧录)
5. 杜邦线若干

二、引脚接线表
------------------------------------------------------------------------
【TFT 显示屏 -> ESP32-S3】
  TFT_CS   ---> GPIO 10  (片选)
  TFT_DC   ---> GPIO 9   (数据/命令选择)
  TFT_RST  ---> GPIO 8   (复位)
  TFT_BL   ---> GPIO 7   (背光，高电平开，低电平关)
  TFT_MOSI ---> GPIO 11  (SPI 数据)
  TFT_SCLK ---> GPIO 12  (SPI 时钟)
  TFT_MISO ---> 不接

  电源:
  VCC  ---> 5V
  GND  ---> GND

【GPS 模块 -> ESP32-S3】
  GPS_TX ---> GPIO 6  (GPS 发送 -> ESP32 接收)
  GPS_RX ---> GPIO 5  (GPS 接收 -> ESP32 发送，可悬空如果不需要串口通信)

  电源:
  VCC  ---> 3.3V 或 5V (根据模块规格)
  GND  ---> GND

------------------------------------------------------------------------
【接线示意图】

  ESP32-S3                     TFT 显示屏              GPS 模块
  ========                     ==========              ========
  GPIO 10  ----------------->  CS
  GPIO 9   ----------------->  DC
  GPIO 8   ----------------->  RST
  GPIO 7   ----------------->  BL (背光)
  GPIO 11  ----------------->  MOSI
  GPIO 12  ----------------->  SCLK
  5V     ----------------->  VCC
  GND    ----------------->  GND

  GPIO 6   <-----------------  TX (注意方向!)
  GPIO 5   ----------------->  RX
  3.3V/5V  ----------------->  VCC
  GND    ----------------->  GND

四、编译烧录
------------------------------------------------------------------------
1. 安装 PlatformIO (VS Code 插件 或 独立 CLI)
2. 打开项目目录 my_esp32_project
3. 编译: pio run
4. 烧录: pio run --target upload (端口 COM8, 可在 platformio.ini 修改)
5. 监视串口: pio device monitor (115200 波特率)

五、WiFi 信息
------------------------------------------------------------------------
设备启动后会自动创建一个 WiFi AP:
  SSID:    GPS-Speedometer
  密码:    12345678
  管理页:  http://192.168.1.1

  用手机或电脑连接此 WiFi 后可访问管理页面，查看:
  - 实时速度和卫星信息
  - 单位切换 (km/h <-> mph)
  - 卫星云图
  - 屏幕颜色配置

七、注意事项
------------------------------------------------------------------------
1. 【供电】建议使用 5V/1A 以上的 USB 电源供电。仅靠电脑 USB 口
   供电可能不足，导致 GPS 定位慢或屏幕闪烁。

2. 【GPS 信号】GPS 模块需要在开阔天空环境下使用。放在室内或车内
   会导致无法定位或卫星数量极少。首次启动需要 30-60 秒冷启动时间。

3. 【屏幕背光】TFT_BL 连接到 GPIO7，高电平开启背光。如果需要关闭
   背光省电，将该引脚设为低电平。

4. 【内存限制】ESP32-S3 RAM 有限，程序已做内存优化:
   - 仅在速度值变化时才刷新屏幕
   - 使用 char[] 替代 String 类型减少内存碎片
   - 不要频繁修改代码中添加大量 String 操作

5. 【SPI 频率】屏幕 SPI 通信频率设置为 20MHz，如需更稳定可降低
   到 16MHz (修改 main.cpp 中的 SPI_FREQUENCY)。

6. 【GPS 波特率】ATGM336H 默认波特率 9600，如需修改请在 GPS 模块
   上使用串口工具调整，并同步修改代码中的 GPS_BAUD。

7. 【WiFi 冲突】AP 名称 "GPS-Speedometer" 可能在其他设备上也使用，
   如遇连接冲突请修改代码中的 SSID。

8. 【复位】如果屏幕白屏无显示，先检查接线是否正确，然后按复位键
   重新运行。串口监视器会打印初始化信息。

8. 【编译平台】本项目使用 ESP32-S3 芯片，不适用于 ESP32-WROOM
   (32位) 或其他型号开发板。

九、常见问题排查
------------------------------------------------------------------------
Q1: 屏幕全白不显示?
  A: 检查 TFT_CS/DC/RST/MOSI/SCLK 接线是否正确，SPI 频率是否匹配。
     串口打印 "TFT Initialized OK" 表示屏幕初始化成功。

Q2: 速度一直为 0?
  A: GPS 模块未定位到卫星。将设备移到室外开阔地带，等待 30-60 秒。
     检查 GPS_TX 是否正确连接到 ESP32 GPIO6。

Q3: 搜不到 WiFi 信号?
  A: 确认程序已正确烧录并运行。检查串口是否有 "WiFi AP started" 信息。
     重启设备重试。

Q4: 串口无输出?
  A: 确认烧录端口(COM8)与实际连接的端口一致。在设备管理器中查看
     ESP32-S3 对应的 COM 端口号，并修改 platformio.ini 中的 upload_port。

Q5: Web 管理页面打不开?
  A: 确认手机/电脑已连接到 "GPS-Speedometer" WiFi。浏览器中输入
     http://192.168.1.1 访问。尝试更换浏览器或清除缓存。
